PDC favorite architecture

PDC framework is consists of four main components

settings_overscan
filter_list PDC - Engine

Worker of PDC

* Responsible for: * Job launch * Job supervise * Metadata update * Written in PERL * 1 to N systems (ETL servers) can consume jobs
filter_list PDC - Metadata Repository

Brain of PDC application

* Job selection for launching * Controlled by metadata * Logic stored in procedures * Chooses jobs for processing regarding to: * Calendar * Dependency * Priority * Locks * Parallelism control * Calendar * Written in Orale PL/SQL
filter_list PDC - GUI

User interface of PDC application

WEB based application for: * Monitoring * Streams * Jobs * Controlling * Scheduler * Application access right * Job’s processing * Multiple Engine support * Access permissions * Metadata change * Change management
filter_list PDC - Framework Checker

Eyesight of PDC application

* Independent supervising process * Checking of correctness of: * Processing initialization * Scheduler * Engine * Job execution * Job behavior * SNMP alert * Written in PERL
