Process Dispatch Center

Process Dispatch Center (PDC) is controlling framework application. The main purpose of PDC is launching separate jobs in sequence based on their dependencies and priorities.

It utilizes optimally systems resources and uses parallelism as much as possible.
The frameworks logic takes care about optimal launch on job level, while the internal logic of the job itself is invisible to the framework.

star How does it work?

filter_1

Framework runs all jobs in the same manner – using a child process which interprets the content of the job command line – and waits for the job exit code.

In dependency on the exit code the framework considers the job finished either successfully or failed. All necessary information needed by the framework is stored as metadata in Oracle database.
